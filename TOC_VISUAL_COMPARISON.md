## Key Changes Made

### 1. Dynamic ToC Detection
**post.html (Lines 62-64)**
```liquid
{% assign headings = content | scan: '<h[23]' | size %}
<!-- Only add class if headings found -->
<div class="post__container{% if headings > 0 %} post__container--with-toc{% endif %}">
```

### 2. Conditional Rendering
```liquid
{% if headings > 0 %}
  {% include toc.html %}
{% endif %}
```
**Effect**: No ToC box appears if there are no h2/h3 headings.

### 3. Flexible Grid Layout
**_sass/_posts.scss (Lines 406-420)**
```scss
.post__container {
  grid-template-columns: 1fr;  /* Default: full width */
  
  &--with-toc {
    @include breakpoint($large) {
      grid-template-columns: 1fr 280px;  /* 2-column only with ToC */
    }
  }
}
```

### 4. Smart ToC Include
**_includes/toc.html**
- Parses rendered HTML for h2/h3 tags
- Extracts IDs (auto-generated by kramdown)
- Builds nested list structure
- Only renders if headings found

## Technical Architecture

```
Markdown File (.md)
    ↓
Jekyll/Kramdown
    ↓ (converts to HTML + adds IDs)
    ↓
Rendered HTML Content
    ↓
Post Layout (post.html)
    ├─ Count h2/h3 tags: scan: '<h[23]'
    ├─ Add class: post__container--with-toc (if found)
    ├─ Include toc.html (if found)
    └─ Render: post__content div
    ↓
ToC Include (_includes/toc.html)
    ├─ Scan for: <h[23][^>]*id="..."
    ├─ Extract: heading level, ID, title
    ├─ Build: nested <ul><li> structure
    └─ Only render: if headings exist
    ↓
CSS (_sass/_posts.scss)
    ├─ post__container: grid-template-columns: 1fr (no ToC)
    ├─ post__container--with-toc: grid-template-columns: 1fr 280px (with ToC)
    └─ post__toc-container: position: sticky (on large screens)
    ↓
Final Rendered Page
```

## Browser Support & Testing

### Desktop Browsers
- ✓ Chrome/Edge (2-column layout, sticky sidebar)
- ✓ Firefox (2-column layout, sticky sidebar)
- ✓ Safari (2-column layout, sticky sidebar)

### Mobile Browsers
- ✓ iOS Safari (full-width, ToC at top)
- ✓ Chrome Mobile (full-width, ToC at top)
- ✓ Firefox Mobile (full-width, ToC at top)

### Edge Cases
- ✓ Post with no h2/h3 headings → No ToC, full-width content
- ✓ Post with only h2 headings → ToC shows, properly indented
- ✓ Post with mixed h2/h3 → Nested structure, proper hierarchy
- ✓ Very long content → Sticky ToC stays visible while scrolling
- ✓ Viewport resize → Layout adapts smoothly
